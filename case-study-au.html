Case Study - Admissions Platform - 2025

Arellano University Online Admission System

Implemented Build

A Web-Based Student Admission and Admin Processing System

AUJASADMS is a full-stack PHP and MySQL platform that digitizes admissions from account creation to final decision. It supports applicant-side submission and tracking, plus an admin-side review pipeline with approve and reject actions, remarks, document viewing, and report exports in CSV and PDF.

01 - Overview

From Manual Admissions to a Structured Online Workflow

The system was built to simplify admissions for both applicants and school personnel. Instead of handling fragmented paper submissions and manual follow-ups, AUJASADMS centralizes student records, requirements, and decision outcomes in one application.

Primary Users

Applicants and admissions staff, with role-based interfaces for Student and Admin accounts.

Core Goal

Deliver a clear submission-to-decision pipeline with status visibility, document validation, and reporting.

02 - Problem and Solution

Reducing Friction in Submission, Review, and Decision Updates

Problem

Manual admissions workflows are slow, hard to track, and prone to missing requirements. Applicants also lack visibility on application progress and reasons for rejection.

Solution

AUJASADMS introduces account-based submission, online requirement uploads, admin review actions, decision remarks, and filterable reporting with CSV and PDF export.

03 - User Flow

Applicant Journey: Register, Submit, and Track Status

Students create an account, log in, and fill a multi-section admission form that includes personal details, family background, educational background, and course preferences. Required documents are uploaded and validated (PDF, JPG, PNG). After submission, applicants can view their full form, current status (Pending, Approved, Rejected), and admin remarks in their dashboard.

04 - Admin Flow

Admin Workflow: Review, Decide, and Communicate

Admins authenticate through a separate portal and manage applications through status-based queues (total, pending, approved, rejected). Each application can be reviewed in detail, documents can be opened securely, and decisions are processed through approve or reject actions. Rejections include custom remarks; approvals auto-attach a confirmation remark for student visibility.

06 - Outcomes

Operational Gains in Visibility and Decision Handling

The platform improved admissions operations by making application data traceable end to end. Staff can monitor applicant volumes and status counts directly from the dashboard, while students receive timely decision updates with remarks. Report generation also became faster through one-click CSV and PDF export.

07 - Architecture

Technical Stack and Data Model

AUJASADMS runs on a standard XAMPP setup using PHP, MySQL, HTML, CSS, Bootstrap, and JavaScript. The data model includes dedicated tables for admins, users, courses, student applications, and archived entities. Application records store both profile fields and admission requirement file paths, plus status and remarks for review outcomes.

08 - Next Steps

Roadmap for a More Robust Admissions Platform

Planned improvements include stronger form-to-user linking to support multiple applications per account, stricter server-side validation and prepared statements across all filters, notification workflows for status changes, richer analytics dashboards, and cleaner document path standardization for long-term maintenance.
